FrontRow.AI"You bring the music. We build the concert."🏛️ Executive ArchitectureFrontRow.AI is a private, ambient venue simulator and experiential presentation layer designed to bring the raw thrill of a front-row live event directly to the user's home. Built specifically for the Musicathon 2026 competition window, the platform solves a massive real-world consumer pain point: providing a definitive, high-fidelity alternative for music fans who love live shows but cannot afford rising ticket prices, travel expenses, or high-density stadium crowds.🛡️ The Intellectual Property DirectiveZero-Audio-Hosting Architecture: FrontRow.AI strictly serves as an experience, visualization, and commentary layer. The application never hosts, serves, streams, or processes copyrighted audio files.Compliance by Design: Licensing liabilities and streaming rights entanglements are entirely eliminated. The user supplies their own acoustic or streaming environment in their room (e.g., physical speakers, record players, local audio), and FrontRow.AI orchestrates a reactive visual stadium backdrop and virtual Master of Ceremonies (MC) commentary wrapped completely around that performance track.🛠️ Monorepo System ArchitectureThe repository utilizes a strict, performance-optimized workspace layout designed to run seamlessly on consumer-grade hardware (such as an Intel i5 configuration) via decoupled client-server orchestration. No local model execution occurs on the client hardware; all operations route dynamically through remote, dedicated partner APIs.├── artifacts/

Unified Arena Search OpenerThe default system interface rejects generic music-app patterns in favor of a raw, performative question: "WHAT SHOW ARE WE CRASHING TONIGHT?" Users are actively guided to input any artist name, song, or upcoming tour target to begin building the stage profile.2. Full-Screen Karaoke Typography FlowManual line-by-line click mechanics are entirely abolished. The client application renders the complete song lyric block simultaneously. Leveraging precise time-sync matrices, the active timed line is heavily illuminated in high-contrast text matching the stage glow, while previous and upcoming lyrics cleanly fade back into the soft ambient shadows of the venue.3. Dynamic Real-World Venue ExtractionThe application extracts venue metadata via the JamBase API proxy. It evaluates the location profile text to dynamically classify the concert into one of five custom, hardware-accelerated CSS/SVG staging layouts:INDOOR_SHOW (Intimate club atmosphere)OUTDOOR_SHOW (Massive festival open lawns)AMPHITHEATER (Layered outdoor concrete stadium beams, scaffolding rigging lines, and ghosted crowd horizons)SYMPHONY_THEATER (High-end, artsy architectural textures)FAIR_LAYOUT (Large-scale local or fairground layouts)4. Non-Blocking Virtual MC HostWhen the arena state shifts to live, the automated Master of Ceremonies dynamically generates a randomized, high-hype introduction script customized to the artist and venue context. Operating via a server-side binary-to-Base64 in-memory transposer, audio tracks are injected instantly to the client's web-audio interface. The announcer runs concurrently as an enhancement layer, meaning lyrics and stage visuals render immediately without waiting on audio buffering barriers.5. High-Fidelity Robust Fallback ProtocolIf premium synchronized telemetry is missing or unavailable for an independent underground track, the server gracefully intercepts the query.
